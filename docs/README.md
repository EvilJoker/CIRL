# CIRL 文档中心

## 文档结构

本项目的文档采用三层结构，遵循"轻、短、准"的写作原则：

### 1. 方向层（稳定）

**位置**：`docs/design.md`

设计总纲，包含：
- 目标与范围
- 统一数据模型
- 技术选型
- 整体框架
- UI 信息架构
- 数据策略

**原则**：只保留稳定结论与承诺，不频繁变更。

### 2. 方案层（可变）

**位置**：`docs/rfcs/`

- `RFC-001-应用优化平台方案.md` - 当前 App 粒度方案快照

**格式**：
- 状态：Provisional（草案）→ Stable（稳定）
- 内容：背景与问题、目标与非目标、具体方案设计、影响面分析

### 3. 实现层（易变）

**位置**：`docs/tech/`

技术实现文档：
- `api-reference.md` - API 参考文档
- `快速开发指南.md` - 开发者快速上手指南

## 当前项目状态

**当前版本**：v1.0.0

**核心功能**：
- 应用管理（App）
- 问答记录跟踪（input/output）
- 反馈闭环处理
- 数据集管理
- 命中分析
- 效果评估
- 优化建议跟踪

**技术栈**：
- 前端：Vue 3 + TypeScript + shadcn-vue（Blue 主题）
- 后端：Node.js + Express
- 数据：SQLite 数据库（默认，支持 JSON Provider 作为备选）

## 文档更新规范

1. **设计总纲**：只更新稳定结论，具体方案以链接方式引用 RFC
2. **RFC 文档**：方案确定后标记为 Stable，重大变更创建新 RFC
3. **技术文档**：随代码更新，保持与实现同步
